{\LARGE \uline{手順}} 

\begin{enumerate}

    \item arpとpingコマンドで, どのプロトコルが利用されるのか調査し, 
          特に通信の開始から終了までに, 通信元と通信先との間に, OSI参照モデルのどの層で, 
          どのような手順で, どのように, どのような情報が, やり取りされるのか詳細を調べ, 整理しなさい. \label{item:arp_ping_investigation}

        \vspace{\baselineskip}

        \quad arpコマンドは, ARP(Address Resolution Protocol)によって, ARPテーブルに記録された情報を表示・編集するためのコマンドである\cite{bashdo_arp}.
        ARPとは, OSI参照モデルの第2層（データリンク層）と第3層（ネットワーク層）の間で動作するプロトコルである.
        ネットワークがコンピュータのIPアドレスと物理的なマシンアドレスを変換（解決）するために使用される一般的な手法であり\cite{kinsta_arp}, 
        宛先ホストのIPアドレスを手がかりに, 次に送るべき機器のMACアドレスを調べる.
        送信元IPアドレスと宛先IPアドレスを比較して, 宛先IPアドレスが同じネットワーク上にあるときには, その宛先IPアドレスのMACアドレスを調べ, 
        宛先IPアドレスが異なるネットワーク上にある場合には, 次に送るルータのIPアドレスからMACアドレスを調べる.
        ARPパケットのフォーマットと各フィールドの説明をそれぞれ図\ref{fig:arp_packet_format}と表\ref{tab:arp_fields}に示す.

            \label{fig:arp_packet_format}
        \end{figure}

        \quad 続いて, ARPによってアドレス解決を行う際の手順について示す.
        ここではホストAがホストBのMACアドレスを解決しようとしているものとする.

        \vspace{\baselineskip}

        \begin{enumerate}[label=\textcircled{\scriptsize\arabic*}]
            \item ホストAは探索するIPアドレスにホストBのIPアドレスを指定したARP要求パケットをネットワークにブロードキャストする.
            \item ホスト B は ARP 要求パケットを受け取ると, ホスト A に対して自身の NIC に割り当てられている MAC アドレスを ARP ユニキャストする.
            \item ホスト A は ARP 応答パケットを受け取り, そこに記述される MAC アドレスはコンピュータ内の ARP テーブルに登録され, 数分間キャッシュされる. 一度登録されるとキャッシュされている間は ARP の再問い合わせが実行されることはない.
        \end{enumerate}

        \vspace{\baselineskip}

        \quad
        pingコマンドとはネットワークの疎通確認やサーバの死活監視をするために使用するコマンドである\cite{knowledgestare_ping}.
        pingを実行すると, ICMPプロトコルは, 指定されたホストにデータグラムを送って, 応答を求める\cite{solaris_ping}.
        ICMP(Internet Control Message Protocol)とはOSI参照モデルの第3層（ネットワーク層）で動作するプロトコルであり,
        IPで通信するときに用いられる. 
        ネットワークが正常な状態であるかを確認することや, 
        異常が発生したときにその状況を把握してトラブルシューティングを行うことなどのために利用するものである. 
        IPパケットのデータ部に図\ref{fig:icmp_packet_format}に示すようなフォーマットで格納される.

        チェックサムとはICMPヘッダーを含むICMPメッセージ全体の誤りをチェックするためのもので,
        データはタイプごとの必要な情報を記述する部分であり, そのフォーマットはタイプによって規定される\cite{nw_text_r08}.

        \end{figure}
        \vspace{\baselineskip}


    \item 実験で得た結果（コマンド実行結果，キャプチャした結果など）を用いて，arp と ping を用いた通信の仕組み・手続きをレポートで報告しなさい．
    
        \vspace{\baselineskip}
        \quad
        手順\ref{item:check_arp_packet}, \ref{item:check_ping_packet}では
        それぞれARPパケットやARPによるアドレス解決の手順とICMPパケットやpingによる通信の手順について,
        実際に得られたパケットの内容を手順\ref{item:arp_ping_investigation}に記した調査結果と比較を行い, 
        想定される手続きでやり取りされていることを確認した.
        ここではまず, 本課題においてpingコマンドを実行した際に実際に行われた通信の流れをまとめ, 最後にarpとpingを用いた通信の仕組み・手続きをまとめる.
        \\
        \quad
        手順10においてserverからrouterへpingコマンドを実行すると, 
        まずARPによるアドレス解決が行われた.
        これはICMPパケットを送信するためには, 送信先のMACアドレスを知る必要があるためである.
        ARPキャッシュに対象のネットワークインターフェース情報が登録されている場合にはARPによるアドレス解決は行われないが, 
        今回の実験では手順6においてARPキャッシュをクリアしたため,
        ARPによるアドレス解決が行われた.
        ここでは, 図\ref{fig:arp_request_packet}のARP要求パケットと図\ref{fig:arp_reply_packet}のARP応答パケットに示すように,
        まずrouterのMACアドレスを求めるためのARP要求パケットがserverからブロードキャストされ, 
        それを受け取ったrouterはARP応答パケットをserverにユニキャストし, 
        最終的にserverのARPキャッシュにrouterのネットワークインターフェース情報が登録された.
        アドレス解決が完了すると, 
        図\ref{fig:icmp_echo_request_packet}のICMPエコー要求パケットと図\ref{fig:icmp_echo_reply_packet}のICMPエコー応答パケットに示すように,
        serverからrouterに対してエコー要求のICMPパケットが送信され,
        routerはエコー要求を受け取るとICMPヘッダにおけるタイプをエコー応答に書き換えたICMPパケットをserverに返送した.

        \quad
        したがって, arpとpingを用いた通信の仕組み・手続きを簡単にまとめると以下のようになる.


        \vspace{\baselineskip}
        \begin{enumerate}[label=\textcircled{\scriptsize\arabic*}]
            \item ARP要求パケットをブロードキャストして宛先MACアドレスを問い合わせる
            \item ARP応答パケットを受信してARPキャッシュに登録する
            \item ICMPエコー要求パケットを送信する
            \item ICMPエコー応答パケットを受信する
        \end{enumerate}
\end{enumerate}
